# Rachel Sargent — bio site

A small, elegant bio site for a writer & editor. It's a fully static page:
`build.py` renders `index.html` from `content.json` at build time, so the served
HTML is complete (great for SEO, no content pop-in, works over `file://`).
Styled with Tailwind (via CDN) and deployed to GitHub Pages.

**Live site:** https://nolaneo.github.io/rach-site/

## Editing content

All the words and books live in [`content.json`](content.json). Non-technical
editing instructions are in [`EDITING.md`](EDITING.md). Edit the JSON, commit,
and the GitHub Action rebuilds and redeploys automatically.

## How it works

`content.json` (data) + `template.html` (structure) → **`build.py`** →
`index.html` (fully-rendered, static). There is no client-side data fetching;
`js/main.js` only handles the section nav and scroll reveals.

```
content.json          # ← all editable content (bio, books, writing, awards)
template.html         # page structure with {{PLACEHOLDERS}}
build.py              # renders content.json + template.html → index.html
dev.py                # local dev server: auto-rebuild + live-reload
css/styles.css        # custom polish (fonts, grain, animations, book cards)
js/main.js            # section nav (scroll-spy/show-hide) + scroll reveals
images/portrait.jpg   # portrait
images/covers/        # book cover art
index.html            # GENERATED by build.py — do not edit (gitignored)
.github/workflows/deploy.yml   # builds + deploys on push to main
EDITING.md            # non-technical editing guide
```

Content is grouped into four editable lists in `content.json`: `booksEdited`,
`booksCoedited`, `writing`, and `awards` — each rendered as its own page section.

## Running locally

**Live preview with auto-reload (recommended):**

```sh
python3 dev.py
# open http://localhost:8000 — edits to content.json/template/css/js
# rebuild and refresh the browser automatically
```

**One-off build:**

```sh
python3 build.py     # writes index.html
open index.html      # works directly over file:// (no server needed)
```

Both use only the Python 3 standard library — no dependencies, no framework.

## Design notes

Editorial / literary aesthetic: a soft blue-grey "paper" background, deep
navy-charcoal ink, a slate-blue accent, hairline rules, and a Fraunces (display)
+ Newsreader (body) type pairing. Book covers are shown uncropped (`object-fit:
contain`) so the colourful art pops against the muted palette. A floating
bottom section-nav jumps between sections. Fully responsive across mobile and
desktop.

## Tech choices

- **Static build, no framework** — a ~40-line, zero-dependency Python generator
  (not React/Astro/etc.). Content ships in the HTML; the page is indexable and
  has no runtime fetch.
- **Tailwind via CDN** — utility styling without a toolchain.
- **GitHub Pages via Actions** — every push to `main` rebuilds and redeploys.
